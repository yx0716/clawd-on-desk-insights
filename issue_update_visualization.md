## 背景

当前 Clawd 的“Check for Updates”主要通过 Electron 原生 `dialog` / `Notification` 呈现更新状态。对这个项目来说，这种交互方式功能上可用，但和桌宠本体的表现是割裂的：用户触发更新后，桌宠本身没有明显参与感，也没有把“检查中 / 下载中 / 失败 / 成功”这些过程映射成可视化动作。

## 建议

希望把更新流程做成和桌宠动作、气泡系统一致的交互，而不是只弹原生更新框。

### 1. 检查更新时

- 桌宠进入类似“侦探巡逻 / 搜查”的动作
- 现有动画里可以直接复用 `sweeping`
- 同时显示自建更新气泡，而不是 Electron 原生弹窗
- 气泡内容显示“正在检查更新”

### 2. 下载更新时

- 桌宠进入“搬运更新包”的动作
- 现有动画里可以复用 `carrying`
- 更新气泡显示“正在下载更新”或“正在更新中”

### 3. 更新失败时

- 桌宠进入 `error` 状态，显示现有 smoke / ERROR 动画
- 更新气泡里直接展示失败原因，而不是只有笼统的失败提示
- 例如网络错误、GitHub API 超时、dirty worktree、git pull 失败、npm install 失败等，都尽量透出实际错误信息

### 4. 更新成功时

- 下载完成或源码更新完成后，桌宠进入 happy / attention 状态
- 更新气泡显示“更新已完成”以及下一步动作，例如：
  - Windows 安装包：`Restart Now` / `Later`
  - macOS：打开下载页
  - Git 模式：提示已完成并即将重启，或展示完成信息

## 希望的交互方向

- 不只是把原生更新弹窗换成一个自定义 bubble
- 而是把“更新流程”整体做成桌宠的一部分
- 让桌宠动作和更新阶段保持一致
- 交互风格尽量和现有权限气泡 / 状态机统一

## 参考映射

- checking -> `sweeping`
- downloading / updating -> `carrying`
- error -> `error`
- success / ready -> `attention`

## 预期收益

- 更新流程和 Clawd 本体表现统一
- 比原生 dialog 更符合桌宠产品形态
- 用户能更直观地理解“现在是在检查、下载、失败还是完成”
- 后续如果加入自动检查，也更容易延展成一致的视觉反馈
